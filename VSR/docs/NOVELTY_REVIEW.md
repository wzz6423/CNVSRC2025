# RSP-VSR 创新最近邻审计

> 检索截止：2026-07-17。范围包括 VSR 说话人适配、持续测试时适配（CTTA）、动态
> adapter/专家、伪标签可靠性、模型恢复/参数恢复以及运动建模。结论基于论文原文、论文主页
> 和作者官方代码；“没有检索到”不等于数学意义上的“从未有人做过”。

## 结论

当前方案**有应用论文潜力，但方法创新风险偏高**。效果好是必要条件，不是充分条件：

- 面向中文连续 VSR，在未知、重复变化的说话人和采集条件下做无源在线个性化，公开研究仍少，
  应用问题本身有价值。
- “冻结主干 + 轻量适配模块”已有 EcoTTA/ViDA；“检测未知域 + 动态分配参数块”已有
  PAINT；“在线域检测 + 可扩展专家 + 重复域回访”已有共享/域自适应专家 + FDD。不能把 adapter、专家增长、
  原型路由或抗遗忘单独写成首创。
- 当前较可能成立的贡献是一个 **VSR 专用安全闭环**：运动感知序列签名、偏移证据累积期的
  隔离、函数保持的新专家、CTC/AED 可靠性判定，以及候选更新级参数与优化器事务回滚。
- 若只证明最终 CER 更低，方法型论文仍容易被评价为组件组合；若能证明隔离机制降低误增长、
  回滚准确识别有害更新、并在真实重复偏移流上优于 PAINT 和共享/域自适应专家，应用型或多媒体/语音方向论文
  才有较扎实的论证。

## 当前实现与已有工作

| 当前组件 | 最近邻证据 | 审计结论 |
| --- | --- | --- |
| 冻结 VSR 主干，只更新残差 bottleneck adapter | [EcoTTA, CVPR 2023](https://arxiv.org/abs/2303.01904) 冻结原网络、只更新轻量 meta networks；[ViDA, ICLR 2024](https://arxiv.org/abs/2306.04344) 使用高/低秩域 adapter | 已有，不可作为独立创新 |
| 余弦原型路由，低相似度时创建新专家 | [PAINT](https://arxiv.org/abs/2412.09308) 用冻结特征、余弦 query/key 和阈值判断未知域并动态分配新 prompt（[代码](https://github.com/Cadezzyr/PAINT)） | 高度重合；adapter 替换 prompt 不构成充分差异 |
| 在线域检测、动态扩展域专家、旧域回访复用 | [Shared & Domain Self-Adaptive Experts/FDD, AAAI 2026](https://arxiv.org/abs/2507.00502) 已有可扩展域专家、无需训练的域检测和 `rain -> sun -> rain` 重复偏移协议（[代码](https://github.com/ZJC25127/Domain-Self-Adaptive-CTTA)） | 最接近的结构最近邻，必须作为主对照 |
| 固定专家池的输入相关路由 | [BECoTTA, ICML 2024](https://arxiv.org/abs/2402.08712) 以低秩域专家和 domain-adaptive routing 做 CTTA（[项目页](https://becotta-ctta.github.io/)） | 专家路由本身已有；当前差异是按需增长与隔离 |
| 连续相似偏移先隔离，达到 patience 才增长 | PAINT 和共享/域自适应专家方法都按当前批次的未知域判定分配参数；在所审最近邻中未见与当前实现相同的“隔离期间禁止更新旧专家” | 可作为组合中的核心差异，但必须证明不是简单平滑阈值 |
| 置信度、多视图一致性和伪标签过滤 | [EATA, ICML 2022](https://arxiv.org/abs/2204.02610)、[PLF](https://arxiv.org/abs/2406.02609) 和 [CUI](https://arxiv.org/abs/2502.02998) 都筛选可靠样本/伪标签 | 已有，不可笼统宣称“可靠性门控首创” |
| 干净/扰动视图的 CTC posterior 一致性 | [CR-CTC, ICLR 2025](https://arxiv.org/abs/2410.05101) 已在 CTC 语音识别中使用跨视图一致性 | 换到 VSR 在线适配是场景迁移，不是损失创新 |
| 更新后验证并恢复 adapter 与优化器状态 | [AETTA, CVPR 2024](https://arxiv.org/abs/2404.01351) 用无标签精度估计触发模型 reset（[代码](https://github.com/taeckyung/AETTA)）；[PETAL, CVPR 2023](https://arxiv.org/abs/2212.09713) 按参数重要性恢复源权重（[代码](https://github.com/dhanajitb/petal)） | 模型恢复已有；当前“每个候选更新的原子回滚”粒度不同，可作为安全机制差异，不能称 TTA 首个 rollback |
| 一、二阶时间差分统计用于路由 | VSR 已长期利用运动信息，如 [双流灰度/光流唇读](https://arxiv.org/abs/1905.02540)；最新 [TVTA](https://arxiv.org/abs/2607.08236) 也强调轨迹/差分运动建模 | 运动特征不新；把无参数运动统计专用于未知域路由，在所审工作中未见完全相同实现 |
| 少量参数做说话人个性化 | [User-dependent Padding, ECCV 2022](https://arxiv.org/abs/2208.04498)、[VSR Prompt Tuning, TPAMI](https://arxiv.org/abs/2302.08102)、[Personalized Lip Reading](https://arxiv.org/abs/2409.00986) 已覆盖无监督 padding、prompt 与 LoRA（[代码](https://github.com/JeongHun0716/Personalized-Lip-Reading)） | “参数高效 VSR 个性化”已有；当前差异必须落在持续、未知且重复的多域流 |

补充边界：[AVReCAP](https://arxiv.org/abs/2602.18528) 已研究 audio-visual CTTA 和历史参数检索，
但任务是 VGGSound 等音视频事件分类，不是连续 VSR。它说明“音视频 CTTA 首创”不能说，
但不直接覆盖本项目的 CTC/Attention 唇读问题。用户提供的 Cued-Bridge 目前是私有代码且没有可核验
论文，能作为工程启发，不能作为新颖性证据或论文贡献来源。

## 可能成立的组合贡献

最稳妥的论文主张不是 “Structural Plasticity is new”，而是：

> 面向无域标签的连续 VSR，提出一种 quarantine-before-grow 的安全在线个性化机制：利用冻结
> 序列特征的运动签名累积偏移证据，避免在域身份尚不确定时污染旧专家；确认后创建函数保持
> adapter；再以 CTC/AED 和 emission 状态验证候选更新，并原子接受或回滚。

这条主张成立需要同时证明三件事：

1. **运动签名确实检测域而不是句子内容。** 当前均值/方差/时间差分可能按音素、语速或句长聚类；
   必须在控制文本内容后仍能识别说话人/设备/姿态偏移。
2. **隔离是独立有效机制。** 相对 PAINT/FDD 式单阈值立即增长，它应显著减少瞬时扰动导致的
   误增长，同时不把适应延迟推高到抵消 CER 收益。
3. **事务判定能预测真实风险。** 同一伪标签、同一样本上的 loss/reliability 改善可能只是自拟合；
   必须证明被回滚的更新在后续窗口中更可能增加真实 CER，而非只证明代码能恢复参数。

如果这三点不成立，剩余内容更适合定位为工程系统或应用报告。若希望冲击方法型 venue，需要把
当前启发式阈值进一步统一成可校准的顺序偏移/风险检验，并给出误增长率或有害更新率的可验证控制，
否则与 PAINT 和共享/域自适应专家方法的距离过近。

## 不能过度宣称

- 不写“首个动态 VSR adapter/MoE”“首个自动专家增长”“首个可靠伪标签 TTA”或“首个 TTA 回滚”。
- 不写“全局最强中文 VSR 模型”；当前只能称所采用官方仓库中已报告的强 checkpoint。
- 不把 motion order 1/2 称为新的动态特征；可称“面向 CTTA 路由的轻量运动签名”。
- 不把零初始化称为理论上的无遗忘保证；它只保证创建瞬间 adapter 输出为恒等，后续更新仍会退化。
- 不以“没有修改主干”推导隐私、安全或端侧可用；还需报告原始视频留存策略、显存、延迟和能耗。
- 不把当前 CNVSRC-Multi 结果自动称为未见域结果；必须按 checkpoint 的训练数据范围排除泄漏。
- 不以单次运行或最终 CER 证明持续学习；必须使用更新前的 prequential CER、回访遗忘和适应延迟。
- 不承诺“效果够好就一定能发表”。强效果能支撑应用论文，但不能消除与最近邻方法的重合。

## 最低实验门槛

以下是投稿前的 go/no-go 条件，不是领域统一标准。

### 对照公平性

- 同一 CNVSRC 基座、同一数据顺序、同一无域标签条件和同等可训练参数/更新 FLOPs，对比静态、
  单 adapter、TENT、EATA、SAR、CoTTA/RoTTA，以及参数预算匹配的 EcoTTA、PAINT、共享/域自适应专家。
- 加入 VSR 专用 prompt/LoRA 说话人适配基线；需要标签或源数据预热的方法必须单独标注资源差异。
- PAINT 和共享/域自适应专家方法不能只引用原论文数字，必须移植到相同 VSR backbone 和流协议。

### 场景与数据

- 至少覆盖未知说话人、设备/相机、姿态/遮挡三类偏移；同时包含 abrupt、gradual、混合相关流和
  `A -> B -> C -> A` 回访流。
- 主结果至少包含一个真实采集集，而非只有亮度、压缩等合成 corruption。建议最低为 20 个未见
  说话人、2 种设备/采集条件、2 个分离 session，并公开采集和划分规则。
- 每条样本先预测并记录，再允许更新；调参流、测试流和反馈计划严格分离。

### 机制证据

- 路由：报告未知域检测 AUROC/F1、路由 purity、误增长率、专家复用率和专家数；按相同文本、不同
  说话人与不同文本、同一说话人分别分析，排除内容聚类。
- 隔离：比较 `growth_patience=1`、仅滑动平均、完整 quarantine；报告误触发与适应延迟的 Pareto 曲线。
- 运动：比较 order 0/1/2、仅 appearance、光流或频域签名；不能只展示最终 CER。
- 回滚：以“该更新对后续固定窗口真实 CER 的影响”为离线真值，报告有害更新检测 precision/recall、
  错杀率和净 CER 收益；分别消融 loss、anchor KL、可靠性、emission 及优化器状态恢复。
- 可靠性：记录伪标签 CER 与接受率的 reliability diagram/ECE，并在 10%/20% 反馈污染和低置信
  伪标签下测试。

### 统计与应用价值

- 至少 3 个随机种子，主比较使用相同流顺序并报告配对 bootstrap 95% CI；建议主结果达到至少
  1 个绝对 CER 点或 5% 相对改善，且置信区间不跨 0。
- 在至少两类独立真实偏移上优于最强最近邻；回访遗忘显著优于单 adapter、PAINT 和共享/域自适应专家。
- 完整报告可训练参数、专家存储、峰值显存、单样本更新延迟、推理延迟和长期专家增长曲线。
- 若完整方法不能同时赢得 prequential CER、回访遗忘和污染安全性中的至少两项，或关键收益仅来自
  一个已有组件，应停止“新方法”投稿定位，转向应用系统或删除无效模块。
